mysql ibdata 太大

Mon 07 December 2015

ibdata 太大无法缩小

  1. Do a mysqldump of all databases, procedures, triggers etc except the mysql and performance_schema databases
  2. Drop all databases except the above 2 databases drop schema xxxxx
  3. Stop mysql service mysqld stop
  4. Delete ibdata1 and ib_log files
  5. Start mysql service mysqld start
  6. Restore from dump

redhat6 sftp 设置

Thu 03 December 2015

创建用户组

用于与sftp设置进行匹配

groupadd  sftp_users

设置sftp

# Subsystem sftp    /usr/libexec/openssh/sftp-server
Subsystem       sftp    internal-sftp
Match Group sftp_users
        X11Forwarding no
        AllowTcpForwarding no
        ChrootDirectory %h
        ForceCommand internal-sftp
  • Match Group sftp_users – 匹配sftp_users组中的用户
  • ChrootDirectory %h – 该参数指定用户验证后用于chroot环境的路径，用户登录后只能查看到该目录下内容，此处设置成你home目录
  • ForceCommand internal-sftp – 该参数强制执行内部sftp
